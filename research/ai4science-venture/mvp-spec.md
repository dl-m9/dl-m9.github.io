# 首个 MVP 与数据规范

## 产品定义

产品名暂定为“Research Loop”。它是一个面向材料或化学研发人员的决策支持系统，不是自动实验控制器。用户给出研发目标、约束、历史实验和批准的工具后，系统返回一批可人工审核的实验建议及其证据、预算和风险说明。

## V1 范围

### 目标任务

在一个固定任务中，从经批准的候选空间内排序下一批 8 到 24 个实验。推荐首个任务选择连续实验、属性可测且有明确成本的配方或工艺优化任务。

### 输入

| 对象 | 最小字段 |
| --- | --- |
| `research_goal` | 目标性质、目标区间、优先级、项目截止时间。 |
| `constraints` | 成本上限、可用原料、工艺边界、禁用项、安全等级、实验批量。 |
| `historical_experiment` | 时间、输入配置、工艺、测量结果、失败原因、成本、耗时、数据来源。 |
| `approved_tool` | 工具名称、允许的参数范围、每次调用成本、输出类型、权限级别。 |
| `evidence_document` | 文献或项目文档的来源、许可状态、摘要、引用定位和可信度。 |

### 输出

| 对象 | 最小字段 |
| --- | --- |
| `recommendation_batch` | 8 到 24 个候选实验、优先级、每项预期信息增益、预测目标、成本和风险。 |
| `evidence_packet` | 每条关键依据的来源、定位、适用条件、反证和不确定性。 |
| `decision_record` | Agent 轨迹摘要、工具调用、预算使用、模型版本和人工审批状态。 |
| `review_queue` | 需要领域专家确认的假设、越界条件和高风险项。 |

## 受控学习闭环

1. 所有建议先保存为不可变的 `decision_record`。
2. 只有人工批准且真实结果回传的项目可以形成 `outcome_record`。
3. V1 只使用历史回放和离线排序评估。模型权重和策略不会因单次前瞻性实验自动更新。
4. 达到预设样本量、误差和安全门槛后，才允许在隔离环境中进行离线后训练。
5. 任何新策略先通过时间切分回放、专家审核和小流量人机协同验证，再进入生产推荐。

## 奖励与评测

早期不将“最终性能”作为唯一奖励。每个实验轨迹至少记录：

- 是否满足硬约束。
- 相对基线的目标性质改善。
- 对不确定性或候选空间的缩减。
- 实验、工具和人工审核成本。
- 失败的可解释原因。
- 证据是否支持建议，是否被专家接受或改写。

离线回放必须按时间切分。某一时点的策略只可使用此前可获得的数据，严禁使用后续实验结果、最终项目结论或未来文献作为输入。

## 安全、权限与 IP

- 系统不得自行下发实验、采购、仪器控制或外部通信动作。
- 高风险化学、受监管生物和未批准工具调用必须进入人工审核队列。
- 客户数据按项目隔离。每个数据源、提示、工具和模型更新都有数据许可与访问日志。
- 联合研发协议必须先定义历史数据的用途、结果数据的训练权、模型改进权、项目 IP 和离开合作后的数据保留期限。
- 没有来源的结论、超出证据适用范围的外推和无法解释的工具结果必须显式标记为不确定，不能包装成推荐事实。

## 验收标准

| 验收项 | 通过标准 |
| --- | --- |
| 证据审计 | 每个关键结论可定位到批准的数据、文献或工具输出。 |
| 数据泄漏 | 时间切分回放中不存在未来结果或未来文献。 |
| 基线比较 | 与专家历史决策、随机策略和当前启发式在同一候选空间比较。 |
| 前瞻性安全 | 所有建议经人工审批，系统无直接实验或采购权限。 |
| 商业结果 | 设计伙伴认可实验数、周期、性能或单位有效发现成本中的至少一项改善指标。 |
