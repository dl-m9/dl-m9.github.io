# AI4Science 创业调研包

更新日期：2026 年 7 月 11 日。

这套材料把“闭环软件加联合研发”的创业假设落成可复核、可访谈、可试点的工作包。它不把公开融资或公司自述当作商业成功，也不代替真实客户访谈、私有数据审查或法律意见。

## 文件导航

| 文件 | 用途 |
| --- | --- |
| [创业决策报告](venture-decision-report.md) | 基于简历、公开案例与赛道比较，给出首选切口和 12 个月路线。 |
| [公司地图](company-landscape.csv) | 五类赛道各 12 家的初筛清单，附官方入口和证据等级。 |
| [证据台账](source-ledger.md) | 报告的公开一手来源、结论边界与核验状态。 |
| [客户访谈与评分](customer-discovery.md) | 20 个访谈的提纲、记录格式、淘汰规则和评分表。 |
| [MVP 与数据规范](mvp-spec.md) | 首个闭环 Agent 的范围、接口、日志、评测和安全边界。 |
| [设计伙伴工作清单](design-partner-worklist.md) | 不含个人联系方式的设计伙伴画像、触达顺序和试点门槛。 |

## 当前结论

首选不是“通用 AI Scientist”，也不是早期自建湿实验室或药物管线。首个可验证方向是面向材料和化学研发团队的**可审计批次实验决策 Agent**：它在给定性能、成本、工艺和安全约束下，利用历史数据、文献和计算工具，为下一批实验排序，并记录实验结果以支持后续受控后训练。

该判断是预访谈结论。只有在至少 3 名同类买方确认痛点、至少 1 个伙伴可提供脱敏历史数据与实验反馈、且 8 至 12 周内能完成结果回放后，才进入付费试点。

## 使用顺序

1. 阅读决策报告，确认切口和不做事项。
2. 从公司地图选择与目标方向最相邻的 10 家公司，按证据台账补做一手核验。
3. 用客户访谈模板完成 20 次访谈。访谈前不要介绍“Agent RL”方案，先确认问题、预算和数据。
4. 只有通过淘汰门槛的场景才进入 MVP 规格和设计伙伴试点。
